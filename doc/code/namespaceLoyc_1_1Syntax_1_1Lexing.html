Enumeration Type Documentation

enum Loyc.Syntax.Lexing.TokenKind

A list of token categories that most programming languages have.

Some Loyc languages will support the concept of a "token literal" which is a tree of tokens, and some DSLs will rely on these token literals for input. However, tokens differ between different languages; for instance the set of operators varies between languages. On the other hand, most languages do have some concept of "an operator", and the TokenKind reflects this fact.

The "TokenKind" concept may assist some simple DSLs to support multiple host languages, by breaking tokens down into common categories. Thus, for instance, all the various strings in a language should be based on TokenKind.String, the dot operator should be TokenKind.Dot, etc. A DSL that just needs simple tokens like "strings", "identifiers" and "dots" can write a parser based on values of Token.Kind alone; if it needs certain specific operators or "keywords" that do not have a dedicated TokenKind, such as + and %, it can further check the Token.Value.
